# Skill: Reporting Work Progress

When the user asks for a report or summary of the work done in this session, follow the structured format below.

## Reporting Format

Use the following 6 sections. Use BOLD and ALL CAPS for headers. Use clear bullet points for details.

### 1. MÔ TẢ CÔNG VIỆC (WHAT)
Describe the main goal of the task and its purpose.

### 2. NỘI DUNG CẦN THỰC HIỆN (HOW)
List the technical approach taken.
- Inherit from existing code or new task
- UI/Logic changes
- Refactoring details

### 3. QUÁ TRÌNH THỰC HIỆN
Detail the actual steps taken and the time spent on each.
- Step 1: Description 
- Step 2: Description 
- ...

### 4. FILE CHANGES
List all files modified, created, or deleted.
- `path/to/file.py` (Action)

### 5. PR
Provide a summary of the Pull Request if applicable, or the branch name.

### 6. SELFTEST + KẾT QUẢ
Detail the verification results.
- Unit tests
- Integration tests
- Screenshots/Logs showing success

## Style Guidelines
- Use **Vietnamese** for the content.
- Use **Bold** for emphasis on key terms.
- Ensure the hierarchy is scannable and professional.
